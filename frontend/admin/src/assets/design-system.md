# Agent Platform 管理后台 — UI 设计规范 v1.0

> 本文档定义 Agent Platform 管理后台的完整设计系统，基于现有 Vue 3 + Element Plus + TypeScript 技术栈，覆盖色彩、布局、组件、页面及交互规范。

---

## 1. 设计原则

| 原则 | 说明 |
|---|---|
| **清晰** | 信息层级明确，用户可快速理解界面结构与当前上下文 |
| **高效** | 核心操作路径不超过 3 步，批量操作和搜索过滤一屏可达 |
| **一致** | 基于 Element Plus 组件库，统一色彩/圆角/间距/字体体系 |
| **可信** | 企业级视觉风格，沉稳专业，适合 AI 平台管理场景 |
| **包容** | 支持桌面端/平板/移动端自适应，支持深色/浅色双主题 |

## 6. 组件设计规范

### 6.1 通用卡片规范

所有页面内容置于 `el-card` 中：

| 属性 | 值 |
|---|---|
| `shadow` | `hover` 或 `never` |
| 圆角 | 8px (header/main 一致) |
| body padding | `20px` |
| header 高度 | 48px，内置 flex 左右布局 |

### 6.2 表格规范

```ts
<el-table stripe highlight-current-row v-loading="loading">
```

| 规范项 | 值 |
|---|---|
| 斑马纹 | `stripe` 启用 |
| 行高 | 44px |
| 每页行数 | 20 (分页器默认) |
| 操作列宽度 | 160-220px，固定右对齐 |
| 日期格式 | `YYYY-MM-DD HH:mm` |
| 空态 | `el-empty` description 统一文案 |

**列宽分配原则：**
- 名称/标题：`min-width="140-180"`
- 描述：`min-width="200"  + show-overflow-tooltip`
- ID / URL：`min-width="240" + show-overflow-tooltip`
- 状态：`width="80-100"`
- 时间：`width="170-180"`
- 操作：`width="160"` 固定右侧

### 6.3 表单规范

| 规范项 | 值 |
|---|---|
| `label-width` | `100px` / `110px` |
| 输入框高度 | 32px (default size) |
| 必填标记 | `required` 属性 |
| 布局 | 单列，复杂表单组件内嵌 `el-row :gutter="16"` |

**表单按钮排序：** 取消 ← 确认/保存 (与 Element Plus 保持一致)

### 6.4 对话框规范

| 规范项 | 值 |
|---|---|
| 宽度 | 新建/编辑 `500-600px`，详情 `800px` |
| 关闭 | `destroy-on-close` 重置表单 |
| 主操作按钮 | `type="primary"`，`loading` 绑定异步 |
| 取消操作 | `@click="show = false"` |

### 6.5 筛选栏规范

首行放置于卡片/页面顶部：

```html
<el-row :gutter="16">
  <el-col :span="8"><el-input v-model="search" clearable /></el-col>
  <el-col :span="4"><el-select v-model="filter" clearable /></el-col>
  <el-col :span="4"><el-select v-model="sort" clearable /></el-col>
</el-row>
```

- 搜索输入框：`span="6-8"`，`clearable` + `prefix-icon="Search"`
- 下拉筛选：`span="3-4"`，`clearable` + `placeholder="状态筛选"`
- 排序/更多：`span="3-4"`
- 总宽度建议 `12-16` 列，剩余留白

### 6.6 空态 & 加载 & 错误

```
┌──────────────────────────┐
│  [Loading Skeleton]      │  → v-loading 全表/卡片加载
│                          │
│  [Empty State]           │  → el-empty + 操作引导
│  暂无数据，点击"创建"开始  │
│                          │
│  [Error State]           │  → el-result status="error"
│  加载失败，点击重试        │
│                          │
│  [Partial Error]         │  → ElMessage.error 弹窗
└──────────────────────────┘
```

**所有列表页初始化时必做：** loading → loaded with data / empty / error。

const signalR = useSignalR()

onMounted(async () => {

## 12. 与现有架构的兼容性

| 现有架构 | 兼容策略 |
|---|---|
| **Pinia stores** | 无需修改 store 接口，只在视图层增强 UI |
| **Axios http.ts** | 保持拦截器、Token 管理、错误处理不变 |
| **Vue Router** | 保持路由结构、导航守卫不变 |
| **Element Plus** | 基于现有组件扩展，避免引入新 UI 库 |
| **SignalR** | Chat 页可选择性替换 SSE 为 SignalR |
| **ECharts** | 保留 Usage 页图表，Dashboard 可新增迷你图 |

**不建议引入的新依赖：**
- 避免引入 Tailwind CSS（与 Element Plus 风格混合增加维护成本）
- 避免替换 Element Plus 内置组件（如 Dialog → Modal 自定义）

**建议引入的轻量依赖：**
- `@vueuse/core` — 响应式工具（`useMediaQuery`, `onClickOutside` 等）
- `vue-json-pretty` — JSON 格式化展示（Skills Schema 预览 / MCP Schema）
- `file-saver` — 导出 CSV/JSON
- `dayjs` — 日期格式化（已内置：Element Plus 依赖）
