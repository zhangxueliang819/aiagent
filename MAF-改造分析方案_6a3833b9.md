# Microsoft Agent Framework 改造分析方案

## 一、Microsoft Agent Framework 1.0 概述

MAF 1.0（2026年4月GA）是 Semantic Kernel + AutoGen 的统一继任者，NuGet 包 `Microsoft.Agents.AI`。核心分层：

```
AIAgent (基类)
├── ChatClientAgent       ← 基于 IChatClient 的标准 Agent
├── CustomAgent           ← 自定义 Agent（继承 AIAgent）
├── A2AAgent              ← A2A 协议代理
└── HarnessAgent          ← 电池全配 Agent（ChatClientAgent + 全套能力）

Workflow Builder          ← 图编排引擎
├── Executors（含 Agent）
├── Edges（条件路由）
├── Checkpoint（状态快照）
└── Human-in-the-Loop

Tools
├── Function Tools        ← 自定义代码工具
├── Code Interpreter      ← 沙箱代码执行
├── File Search           ← 文件搜索
├── Web Search            ← 网络搜索
├── Hosted MCP Tools      ← 服务端 MCP
├── Local MCP Tools       ← 本地 MCP
└── Agent as Tool         ← Agent 作为工具
```

## 二、当前项目 vs MAF 全面差距分析

### 2.1 数据模型对标

| 当前实体 | MAF 对应概念 | 匹配度 | 差距说明 |
|----------|-------------|--------|---------|
| `Agent` | `AIAgent` / `ChatClientAgent` | 40% | 当前是纯数据实体，MAF 的 Agent 是运行时对象。需要重构为：Agent 定义(数据) + Agent 运行时(MAF AIAgent 包装) |
| `Agent.Status` (Draft/Active/Inactive/Archived/Running/Paused/Stopped) | 无内置状态机 | 60% | MAF 不内置业务状态机，但你项目的状态机设计合理，可保留作为管理层面的 Agent 生命周期 |
| `Agent.SystemPrompt` | `ChatOptions.Instructions` | 90% | 概念完全对应，改造时直接映射 |
| `Agent.Temperature/MaxTokens/TopP` | `ChatOptions` 参数 | 90% | 概念对应，但 MAF 通过 `ChatOptions` 传递，而非 Agent 实体属性 |
| `AgentConfiguration` | `AIContextProvider` / `StateBag` | 30% | 当前是 KV 配置表，MAF 使用 Context Provider 模式注入动态上下文 |
| `AgentSkill` (多对多) | Function Tools | 70% | 关联关系正确，但 MAF 的 Tool 是运行时注册的函数，非数据库实体。Skills 表需保留作为"托管工具注册表" |
| `AgentMcpEndpoint` (多对多) | Local MCP Tools / Hosted MCP Tools | 80% | 关联关系正确，MAF 原生支持 MCP，改造后可无缝对接 |
| `AgentVersion` | 无内置版本管理 | 80% | MAF 不提供版本快照，你项目的版本管理是增值功能，保留 |
| `Skill` (Tool/Api/Script/Composite) | Function Tools | 50% | 当前 Skill 类型枚举(Tool/Api/Script/Composite)过于自定，MAF 统一为 Function Tool + Code Interpreter + Web Search + File Search |
| `McpEndpoint` | MCP Server 配置 | 85% | 结构匹配良好，协议字段(SSE/STDIO)可直接复用 |
| `McpTool` | MCP Tool 缓存 | 90% | 完全对应，MAF 的 `LocalMcpTools` 也有工具发现+缓存逻辑 |
| `Session` | `AgentSession` | 50% | 当前是数据实体，MAF 的 Session 是运行时 `StateBag` 容器。需重构为双轨：Session(持久化) + AgentSession(运行时状态) |
| `Conversation` | Chat History | 70% | 基础结构可用，但缺少 ToolCallId/ToolName 等 function calling 字段 |
| `ModelProvider` | `IChatClient` 工厂 | 60% | 当前是数据实体，MAF 中每种 Provider 直接创建对应的 `IChatClient` 实例 |
| `ModelEndpoint` | Provider 端点配置 | 80% | RPM/TPM/Weight 字段良好，可作为 `IChatClient` 的构造参数 |
| `UsageRecord` | OpenTelemetry metrics | 50% | 当前自定义结构，MAF 推荐 OpenTelemetry，但可保留作为业务计费数据 |
| `AuditLog` | 无内置审计 | 100% | MAF 不提供业务审计，保留 |
| `ApiKey` | 无内置 API Key 管理 | 100% | MAF 不提供，保留 |

### 2.2 AgentEngine 运行时对标

| 当前组件 | MAF 对应能力 | 匹配度 | 改造建议 |
|----------|-------------|--------|---------|
| `AgentRuntime.RunAsync()` | `AIAgent.RunAsync()` | 50% | 当前手动实现对话循环 + JSON 解析 function_call。MAF 内置标准 Agent Loop，应废弃自建循环 |
| `AgentRuntime.BuildSystemMessage()` | Harness instructions + Context Providers | 30% | 当前手动拼接 System Prompt + 工具定义文本。MAF 通过 `ChatOptions.Instructions` + `AIContextProvider` 注入，工具 Schema 由框架自动生成 |
| `FunctionCallHandler` | MAF Function Calling (内置) | 20% | 当前手动 JSON 解析匹配函数名。MAF 原生支持 `AIFunctionFactory.Create()` 注册函数，框架自动处理调用分发 |
| `SkillDispatcher` | Function Tools 注册 | 40% | 当前按 SkillType 分派到不同 Executor。MAF 中每种 Skill 应注册为独立的 `AIFunction` |
| `ShortTermMemoryStore` | Session Chat History + Compaction | 40% | 当前基于 Conversation 表实现滑动窗口。MAF 有 `InMemoryChatHistoryProvider` + Compaction 策略 |
| `McpClient` | `LocalMcpTools` / `HostedMcpTools` | 30% | 当前手动实现 MCP 协议。MAF 内置 MCP Client，应直接使用框架能力 |

## 三、MAF Agent Skills 与当前项目技能管理统一方案（核心）

### 3.1 概念澄清：两套"技能"的本质差异

MAF 的 Agent Skills 与当前项目的 Skill 是**两种完全不同的概念**，但它们可以互补统一：

| 维度 | MAF Agent Skills | 当前项目 Skill |
|------|-----------------|---------------|
| **本质** | 知识/指令包（动态 System Prompt 片段） | 函数工具（LLM function calling 目标） |
| **载体** | `SKILL.md` 文件（YAML frontmatter + Markdown 指令） | 数据库实体（Name/Type/Implementation/InputSchema） |
| **注册方式** | 文件系统发现 / 代码内联 / 类派生 / MCP 发现 | 数据库 CRUD + 依赖注入 `ISkillExecutor` |
| **执行模式** | 四阶段渐进式披露：宣告→加载→读资源→跑脚本 | 一次性：LLM function_call → SkillDispatcher → ISkillExecutor |
| **工具集** | `load_skill` / `read_skill_resource` / `run_skill_script` | 自定义 function_call 名称（skill.Name） |
| **与 Agent 关系** | 通过 `AIContextProviders` 注入 | 通过 `AgentSkill` 多对多关联表 |
| **典型用途** | "费用报销流程""代码规范指南""数据分析管道" | "查天气""发邮件""调用 API" |

**结论：MAF Skills = "教 Agent 怎么做的知识包"，项目 Skills = "让 Agent 调用的工具函数"。两者是正交关系，可以共存。**

### 3.4 两种 Skill 类型的运行时映射

#### Type = FunctionTool（LLM 可调用工具）

当前项目已有的 `ApiSkillExecutor` / `ToolSkillExecutor` 直接改造为注册 `AIFunction`：

```csharp
// 新文件: src/AgentPlatform.AgentEngine/Skills/FunctionToolRegistry.cs

public class FunctionToolRegistry
{
    private readonly ISkillRepository _skillRepo;

    /// <summary>
    /// 将数据库中 Type=FunctionTool 的 Skill 转换为 AIFunction 列表，
    /// 供 ChatClientAgent 的 ChatOptions.Tools 使用
    /// </summary>
    public async Task<List<AIFunction>> GetFunctionsForAgentAsync(Guid agentId)
    {
        var skills = await _skillRepo.GetByAgentIdAsync(agentId);
        
        return skills
            .Where(s => s.Type == SkillType.FunctionTool && s.IsEnabled)
            .Select(s => AIFunctionFactory.Create(
                name: s.Name,
                description: s.Description,
                parameters: JsonSchemaParser.Parse(s.InputSchema),
                implementation: async (args) =>
                {
                    // 委托给现有的 ISkillExecutor 执行
                    var result = await _skillDispatcher.ExecuteByNameAsync(s.Name, args);
                    return result.Data ?? "{}";
                }
            ))
            .ToList();
    }
}
```

#### Type = AgentSkill（MAF 知识技能）

将数据库中的 AgentSkill 包装为 MAF `AgentInlineSkill`，注册到 `AgentSkillsProvider`：

```csharp
// 新文件: src/AgentPlatform.AgentEngine/Skills/DatabaseSkillSource.cs

/// <summary>
/// 自定义 MAF SkillSource：从数据库 Skills 表读取 AgentSkill 类型记录，
/// 封装为 AgentInlineSkill 并注入 AgentSkillsProvider
/// </summary>
public class DatabaseSkillSource : AgentSkillsSource
{
    private readonly ISkillRepository _skillRepo;
    private readonly IAgentSkillRepository _agentSkillRepo;

    public DatabaseSkillSource(ISkillRepository skillRepo, IAgentSkillRepository agentSkillRepo)
    {
        _skillRepo = skillRepo;
        _agentSkillRepo = agentSkillRepo;
    }

    public override async Task<IList<AgentSkill>> GetSkillsAsync(
        AgentSkillsSourceContext context, CancellationToken ct)
    {
        // 根据 context.Agent 获取该 Agent 绑定的技能
        var agentId = GetAgentIdFromContext(context);
        var bindings = await _agentSkillRepo.GetByAgentIdAsync(agentId, ct);
        var skillIds = bindings.Where(b => b.IsEnabled).Select(b => b.SkillId).ToList();
        var skills = await _skillRepo.GetByIdsAsync(skillIds, ct);

        var mafSkills = new List<AgentSkill>();
        foreach (var skill in skills.Where(s => s.Type == SkillType.AgentSkill))
        {
            var inlineSkill = new AgentInlineSkill(
                name: skill.Name.ToLowerInvariant().Replace(" ", "-"),
                description: skill.Description,
                instructions: skill.Implementation);  // Implementation 存 Markdown 指令

            // 如果 InputSchema 中有资源定义，附加为资源
            // ... AddResource() ...

            mafSkills.Add(inlineSkill);
        }
        return mafSkills;
    }
}
```

### 3.8 AgentSkill 的三种存储与加载方式

MAF 原生的 Skill 加载是**基于文件系统目录**的，但项目需要支持前端上传和数据库管理。为此设计三种互补的存储方式：

| 方式 | 存储位置 | 前端操作 | 运行时加载 | 适用场景 |
|------|---------|---------|-----------|---------|
| **数据库内联** | `Skill.Implementation` 字段存 Markdown | 在表单中直接编辑 Markdown 正文 | `DatabaseSkillSource` → 转为 `AgentInlineSkill` | 简单的纯文本指令 Skill |
| **文件上传** | 上传 zip → 解压到 `uploaded-skills/{skillId}/` | 拖拽/选择 .zip 文件上传 | `AgentFileSkillsSource(uploaded-skills/)` → MAF 原生加载 | 完整的 SKILL.md 包（含脚本和资源） |
| **目录挂载** | 服务器本地 `skills/` 目录 | 管理员手动放置/CI 部署 | `AgentFileSkillsSource(skills/)` → MAF 原生加载 | 内置系统 Skill、CI 部署的企业 Skill |

这三种方式最终统一汇入 `AgentSkillsProviderBuilder`，实现混合加载。

#### 3.8.1 文件上传流程

```
前端上传 .zip
     │
     ▼
POST /api/v1/skills/upload
     │  multipart/form-data { file: expense-report.zip }
     ▼
SkillsController.UploadAsync()
     │  1. 验证 zip 内是否包含 SKILL.md
     │  2. 解压到 uploaded-skills/{skillId}/
     │  3. 解析 SKILL.md frontmatter → 提取 name, description
     │  4. 写入 Skills 表: StorageType=File, StoragePath=uploaded-skills/{skillId}/
     │  5. 返回 Skill 实体
     ▼
Agent 运行时
     │  AgentSkillsProviderBuilder
     │    .UseFileSkill(uploaded-skills/)    ← MAF 原生目录发现
     │    .Build()
     ▼

```

#### 3.8.3 运行时混合加载逻辑

这是核心——三种来源的技能统一汇入 MAF 的 `AgentSkillsProvider`：

```csharp
// 新文件: src/AgentPlatform.AgentEngine/Skills/UnifiedSkillProviderFactory.cs

public class UnifiedSkillProviderFactory
{
    private readonly ISkillRepository _skillRepo;
    private readonly IAgentSkillRepository _agentSkillRepo;
    private readonly IConfiguration _config;

    /// <summary>
    /// 为指定 Agent 构建统一的 AgentSkillsProvider
    /// 混合三种来源：数据库内联 + 文件上传 + 目录挂载
    /// </summary>
    public async Task<AgentSkillsProvider> CreateForAgentAsync(
        AIAgent agent, Guid agentId, CancellationToken ct)
    {
        var builder = new AgentSkillsProviderBuilder();

        // 1. 加载数据库内联技能（Type=AgentSkill, StorageType=Inline）
        var dbSkills = await GetAgentDbSkillsAsync(agentId, ct);
        foreach (var skill in dbSkills.Where(s => s.StorageType == SkillStorageType.Inline))
        {
            builder.UseSkill(skill.ToInlineAgentSkill());
        }

        // 2. 加载上传的文件技能目录（Type=AgentSkill, StorageType=File）
        //    AgentFileSkillsSource 自动扫描目录下所有 SKILL.md
        var uploadedPaths = dbSkills
            .Where(s => s.StorageType == SkillStorageType.File && s.StoragePath != null)
            .Select(s => s.StoragePath!)
            .ToList();
        if (uploadedPaths.Count > 0)
        {
            builder.UseFileSkill(uploadedPaths);  // ← MAF 原生，自动发现 SKILL.md
        }

        // 3. 加载全局目录挂载技能（服务器本地 skills/ 目录）
        var globalSkillsDir = _config["Skills:GlobalDirectory"] ?? "skills/";
        if (Directory.Exists(globalSkillsDir))
        {
            builder.UseFileSkill(globalSkillsDir);
        }

        // 4. 注入脚本执行器（用于基于文件的技能中的脚本）
        builder.UseFileScriptRunner(SubprocessScriptRunner.RunAsync);

        return builder.Build();
    }
}
```

| `references/` 目录 | zip 内的 references/ 子目录 | `uploaded-skills/{id}/references/` |
| `assets/` 目录 | zip 内的 assets/ 子目录 | `uploaded-skills/{id}/assets/` |
| `AgentFileSkillsSource` | `UnifiedSkillProviderFactory` 中 `.UseFileSkill(uploadedPaths)` | `AgentPlatform.AgentEngine/Skills/` |
| `AgentSkillsProvider` | 由 `UnifiedSkillProviderFactory.CreateForAgentAsync()` 创建 | `AgentPlatform.AgentEngine/Skills/` |
| `SubprocessScriptRunner` | 注入到 `UseFileScriptRunner()` 用于执行 py/js/sh 脚本 | `AgentPlatform.AgentEngine/Skills/` |
| 渐进式披露（宣告→加载→读资源→跑脚本） | 完全由 MAF `AgentSkillsProvider` 自动处理，无需自建 | - |

### 3.9 Skill.Implementation 字段的新语义

| SkillType | Implementation 字段内容 | 示例 |
|-----------|----------------------|------|
| `FunctionTool` | C# 类名（实现了 `ISkillExecutor`，通过 DI 解析） | `WeatherApiExecutor` |
| `AgentSkill` | Markdown 指令正文（即 SKILL.md 的 body 部分） | `"使用此技能处理费用报销：\n1. 验证发票\n2. ..."` |
| `McpTool` | MCP 工具全限定名 | `mcp://server1/tools/get_weather` |

### 3.10 Skill.InputSchema 字段的新语义

| SkillType | InputSchema 内容 |
|-----------|-----------------|
| `FunctionTool` | 函数的 JSON Schema（参数定义，供 LLM function calling 使用） |
| `AgentSkill` | 资源的 JSON 列表（可选附带的 references/assets 元数据） |
| `McpTool` | 空（Schema 从 McpTool 表获取） |

### 3.11 与 Agent 的关联方式

两种 Skill 类型通过同一个 `AgentSkill` 关联表挂载到 Agent，但运行时行为不同：

```
Agent
├── AgentSkill (Priority=1, SkillId=func-weather)
│   └── Skill (Type=FunctionTool) → 注册为 AIFunction → 放入 ChatOptions.Tools
├── AgentSkill (Priority=2, SkillId=skill-expense)
│   └── Skill (Type=AgentSkill)  → 注册为 AgentInlineSkill → 放入 AgentSkillsProvider
└── AgentMcpEndpoint (Priority=3, McpEndpointId=mcp-1)
    └── McpEndpoint → 注册为 LocalMcpTools → 放入 ChatOptions.Tools
```

运行时组装逻辑（新文件 `AgentRuntimeFactory.cs`）：

```csharp
public async Task<AIAgent> CreateAgentAsync(Agent entity, CancellationToken ct)
{
    var chatClient = _modelProviderFactory.CreateChatClient(entity);

    // 1. 获取 FunctionTool 类型技能 → AIFunction 列表
    var functionTools = await _functionToolRegistry.GetFunctionsForAgentAsync(entity.Id, ct);

    // 2. 获取 AgentSkill 类型技能 → AgentSkillsProvider
    var skillsProvider = new AgentSkillsProviderBuilder()
        .UseFileSkill("skills/")                          // 全局文件技能库
        .UseSkillSource(new DatabaseSkillSource(...))      // 数据库中的 AgentSkill
        .Build();

    // 3. 获取 MCP 工具
    var mcpTools = await _mcpToolRegistry.GetMcpToolsForAgentAsync(entity.Id, ct);

    AIAgent agent = chatClient.AsAIAgent(new ChatClientAgentOptions
    {
        Name = entity.Name,
        ChatOptions = new()
        {
            Instructions = entity.SystemPrompt,
            Tools = functionTools.Concat(mcpTools).ToList(),  // LLM 可调用的工具
        },
        AIContextProviders = [skillsProvider],               // MAF Skills 上下文注入
    });

    return agent;
}
```

### 3.13 Skill 表结构最终方案

```sql
-- Skills 表（改造后）
CREATE TABLE Skills (
    Id              UNIQUEIDENTIFIER PRIMARY KEY DEFAULT NEWID(),
    Name            NVARCHAR(200) NOT NULL,
    Description     NVARCHAR(2000),
    SkillType       TINYINT NOT NULL,  -- 0=FunctionTool, 1=AgentSkill, 2=McpTool
    Implementation  NVARCHAR(MAX),     -- FunctionTool: Executor类名; AgentSkill: Markdown指令; McpTool: 工具标识
    InputSchema     NVARCHAR(MAX),     -- FunctionTool: JSON Schema; AgentSkill: 资源清单JSON; McpTool: {}
    IsEnabled       BIT NOT NULL DEFAULT 1,
    CreatedAt       DATETIME2 NOT NULL DEFAULT GETUTCDATE(),
    UpdatedAt       DATETIME2 NOT NULL DEFAULT GETUTCDATE()
);

-- AgentSkills 关联表（不变）
CREATE TABLE AgentSkills (
    Id              UNIQUEIDENTIFIER PRIMARY KEY DEFAULT NEWID(),
    AgentId         UNIQUEIDENTIFIER NOT NULL REFERENCES Agents(Id),
    SkillId         UNIQUEIDENTIFIER NOT NULL REFERENCES Skills(Id),
    Priority        INT NOT NULL DEFAULT 0,  -- FunctionTool: LLM调用优先级; AgentSkill: 宣告顺序
    IsEnabled       BIT NOT NULL DEFAULT 1
);
```

### 3.14 数据迁移路径

```
旧 Type=Tool,Api  →  新 Type=FunctionTool
  - Implementation 保留类名
  - InputSchema 保留不变

旧 Type=Script    →  新 Type=AgentSkill
  - Implementation 改为 Markdown 指令（从原 Implementation 字段的脚本内容改造）
  - InputSchema 改为资源清单 JSON（可选）

  - Implementation 写组合指令的 Markdown
  - 子 Skill 引用写入资源清单
```

## 六、关键设计决策

1. **双轨架构策略**：Agent 定义（数据库实体）≠ Agent 运行时（MAF AIAgent）。Agent 实体存储元数据和配置，运行时通过工厂创建 MAF Agent 实例。
2. **Skills 双轨统一**：MAF Agent Skills（知识指令包）与项目 FunctionTool（函数工具）共存于同一 `Skills` 表，通过 `SkillType` 区分。FunctionTool → `AIFunction`（function calling），AgentSkill → `AgentInlineSkill`（渐进式披露）。
3. **自定义 SkillSource 桥接数据库**：实现 `DatabaseSkillSource`（继承 `AgentSkillsSource`），使 MAF 的 `AgentSkillsProvider` 可以从数据库动态读取 AgentSkill 记录，与基于文件的 SKILL.md 技能无缝混合。
4. **自建 MCP 客户端废弃**：全面使用 MAF 内置的 `LocalMcpTools` / `HostedMcpTools`。
5. **Session 双态设计**：`Session` 实体（持久化）+ `AgentSession.StateBag`（运行时），通过序列化桥接。
